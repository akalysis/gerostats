Building AKALYSIS: from zero to production.

How I designed, engineered and deployed the Akalysis consultancy site using Next.js, TypeScript, Tailwind CSS, Vercel and Resend, and why a statistician building their own production infrastructure matters.

Next.js TypeScript Tailwind CSS Vercel Resend

The Akalysis site was designed, engineered and deployed by me. Not outsourced, not templated, not a drag-and-drop build. It was built from scratch in Next.js, deployed on Vercel with a custom domain, and given a working transactional email pipeline.

Why this belongs in the garden: building production software is part of my skill set, not a side note. A statistician who can also ship clean, performant web infrastructure is a different kind of hire. This note is evidence of that, not just a claim about it.

Stack decisions

Every technology choice was deliberate. The goal was a site that is fast, maintainable, search-friendly and capable of growing without avoidable technical debt.

Next.js with the App Router

React with the App Router gives server-side rendering by default, file-system routing and a clean separation between server and client components. For a content-heavy consultancy site that needs good Core Web Vitals and structured metadata, this is a strong foundation. Pages are rendered at build time where possible, and dynamic routes can be generated statically.

TypeScript throughout

The content layer, SEO definitions, publication lists and garden entries are typed with explicit interfaces. That is not ceremonial. It means a typo in a content object fails at compile time rather than silently producing a broken production page. 

// src/lib/content/seo-pages.ts
export type SeoLandingPageContent = {
  slug: string;
  path: string;
  metaTitle: string;
  intro: string;
  focusAreas: Array<{
    title: string;
    description: string;
  }>;
};

The type system catches structural errors before they reach the browser. That is the same principle behind a good statistical analysis plan: define your schema before you touch the data.

Tailwind CSS and a deliberate design system

The original Akalysis design used a high-contrast dark interface with lime, emerald and teal accents. The point was to avoid a generic blue-heavy technology palette and to make the site feel precise, technical and recognisable at small sizes.

MDX for the digital garden

Long-form garden content was written in MDX: Markdown with embedded React components. The intercensal method post, for example, used rendered LaTeX equations inside an otherwise readable article. That is the right pattern when writing needs notation, code, diagrams or interactive components without losing editorial flow.

The contact pipeline

The contact form is a compact example of lightweight backend work. Rather than relying on a mail client opening correctly, the form posts to a Next.js API route that sends a formatted email via Resend. 

// src/app/api/contact/route.ts
import { Resend } from "resend";
import { NextRequest, NextResponse } from "next/server";

const resend = new Resend(process.env.RESEND_API_KEY);

export async function POST(request: NextRequest) {
  const { name, email, enquiryType, message } = await request.json();

  await resend.emails.send({
    from: "AKALYSIS Enquiries <contact@akalysis.co.uk>",
    to: ["mrakingston@gmail.com"],
    replyTo: email,
    subject: `New enquiry from ${name} - ${enquiryType}`,
    html: `...`
  });

  return NextResponse.json({ success: true });
}

Domain verification, DNS and SSL were handled through Vercel, with sensitive keys kept as environment variables rather than committed to the repository.

Deployment and domain

The site was deployed on Vercel with automatic deployments triggered by pushes to the main branch on GitHub. A local commit could be live within roughly a minute, including build time. 

$ git push origin main
-> GitHub receives push
-> Vercel webhook triggers build
-> Next.js build runs
-> akalysis.co.uk updated

The domain was registered separately, with nameservers pointing to Vercel. That leaves no server to maintain, no certificate renewal to schedule and no hand-rolled hosting layer to babysit.

Why this matters for a statistician

Data scientists who can only operate inside pre-built environments are limited in what they can deliver. The ability to take analytical output and build the infrastructure that surfaces it to decision-makers is a meaningful extension of the role.

Akalysis was a working example of that approach: statistical writing served by a production web application with typed data models, structured metadata, automated deployment and transactional email, all built and maintained by the same person doing the analysis.
